# N-BOM メイン画面 UI/UX 設計案（機器特化版）

**プロジェクト**: プラント積算システム 次世代版（N-BOM）
**対象画面**: メイン画面（T-BOM 画面番号 101a / 101b の統合後継）
**設計方針**: 「フォルダ型管理」から「属性（タグ）型管理」へのパラダイムシフトに対応
**スコープ**: 機器積算に特化（配管資材は対象外）

---

## 1. 現行T-BOMの課題と設計上の対応方針

### 1.1 課題の整理（機器積算に関するもの）

| #   | 課題                   | 構造的要因                                 | 現場への影響                                                        |
| :-- | ---------------------- | ------------------------------------------ | ------------------------------------------------------------------- |
| C1  | フォルダ階層依存       | 属性情報がフォルダ名に埋め込まれた設計     | 担当者ごとの構成不統一、エリア・工種別の再集計に手作業発生、属人化  |
| C2  | 階層数の制約           | 最大4階層のシステム制限                    | 大規模案件で機器の細分化が不能、案件コピー時のデータ不整合          |
| C3  | 実績データのサイロ化   | 過去見積の比較・参照が困難                 | 見積精度の低下、パフォーマンス劣化、PDFが散在し有効活用できていない |
| C4  | 処理パフォーマンス劣化 | テンポラリテーブル作成による対症療法的処理 | データ量増大に伴い集計処理に著しい時間を要する                      |

### 1.2 設計方針マッピング

| 課題                | N-BOM UIの解決アプローチ                                                 |
| ------------------- | ------------------------------------------------------------------------ |
| C1 フォルダ依存 →   | 属性タグベースのフラットデータモデル＋多次元ビュー切替                   |
| C2 階層制約 →       | 階層制限の撤廃。タグによるグルーピングで論理的な「ビュー階層」を動的生成 |
| C3 データサイロ →   | 実績データベースとの統合検索パネル＋過去工番との比較機能                 |
| C4 パフォーマンス → | 仮想スクロール＋インクリメンタル集計による根本的改善                     |

## 2. 画面構成の全体設計

### 2.1 設計コンセプト

現行T-BOMは「ツリー表示（101a）」と「機器別表示（101b）」を**別画面として切り替える**構造だった。N-BOMでは「ビューの軸切替」という概念に統合し、**1つの画面でシームレスに表示軸を切り替えられる**設計とする。

**デザイン原則:**

| 原則                   | 内容                                                                         |
| ---------------------- | ---------------------------------------------------------------------------- |
| Single Source of Truth | 同一の機器データを複数の「見え方」で表示。データモデルは1つ                  |
| Progressive Disclosure | 頻用機能（仕様選定・単価更新等）を前面に、専門機能は段階的に開示             |
| Context-Aware Actions  | 選択中の機器種別（回転機/静機器/計装機器等）に応じてアクション群を動的に変更 |
| Keyboard-First         | ショートカットキーによる高速操作を前提とした設計                             |

### 2.2 全体レイアウト

```text
┌─────────────────────────────────────────────────────────────────┐
│ A. グローバルヘッダー                                           │
│ [N-BOM Logo]  コマンドパレット(⌘K)  [通知] [ユーザー] [設定]   │
├────────┬────────────────────────────────────────────────────────┤
│        │ B. ビューコントロールバー                              │
│        │ [ビュー軸切替タブ] [フィルターバー] [検索] [列設定]    │
│        ├────────────────────────────────────────────────────────┤
│        │                                                        │
│ C.     │ D. メインデータグリッド                                │
│ JOB    │                                                        │
│ ナビ   │ 属性タグベースの階層グリッド                           │
│ ゲー   │ （ビュー軸に応じたグルーピング＋スプレッドシート表示）  │
│ ター   │                                                        │
│        │                                                        │
│        ├────────────────────────────────────────────────────────┤
│        │ E. サマリーバー（選択機器の集計値をリアルタイム表示）   │
├────────┼────────────────────────────────────────────────────────┤
│        │ F. ディテールパネル（右サイドまたは下部に展開）         │
│        │ 選択機器の詳細属性・仕様・履歴・過去実績               │
└────────┴────────────────────────────────────────────────────────┘
```

**レスポンシブ**: 最小横幅 1280px。1920px以上ではディテールパネルを右に常時展開した3カラム構成。

## 3. 各領域の詳細設計

### 3.1 A. グローバルヘッダー

**目的**: システム全体のナビゲーションとクイックアクセス

| 要素              | 説明                                                         | 操作       |
| ----------------- | ------------------------------------------------------------ | ---------- |
| ロゴ / システム名 | 「N-BOM」。クリックでダッシュボードに戻る                    | クリック   |
| コマンドパレット  | `⌘K` / `Ctrl+K` で起動。あらゆる操作をキーワード検索で即実行 | キーボード |
| 通知アイコン      | 他ユーザーからの変更通知、バッチ処理完了通知                 | クリック   |
| ユーザーメニュー  | ログアウト、権限情報、環境設定                               | クリック   |

**コマンドパレットの設計意図**: 現行T-BOMではメニューバーに多数の機能が階層的に配置されており、目的の機能にたどり着くのに平均3クリック以上を要している。コマンドパレットにより「単価更新」「仕様選定」等をタイプするだけで即座に到達可能とする。

```text

│ ─────────────────────                   │
│ すべての操作                            │
│   📝 編集 > 工事情報                    │
│   📝 編集 > 仕様選定                    │
│   ➕ 作成 > 原価項目追加                │
│   📊 積算 > 単価更新                    │
│   📊 積算 > 経費積算                    │
│   📊 積算 > 原価集計                    │
│   🔄 CAD > CAD WorxP&ID取込             │
│   ...                                   │
└─────────────────────────────────────────┘
```

### 3.2 B. ビューコントロールバー

**目的**: T-BOMの「ツリー表示/機器別表示」の二択を拡張し、**任意の属性軸でのグルーピングビュー**を切り替える

**解決する課題**: C1（フォルダ依存）、C2（階層制約）

#### ビュー軸切替タブ

```text
┌──────────────────────────────────────────────────────────────┐
│ [📁 フォルダ] [🏭 機器別] [📍 エリア別] [🔧 工種別]        │
│ [📊 進捗別]  [+ カスタムビュー]                              │
│                                                              │
│ フィルター: [工区 ▾][リストタイプ ▾][メーカ ▾][+ 条件追加]  │
│                                                              │
│ 🔍 機器検索...              [列設定 ⚙] [エクスポート ↓]     │
└──────────────────────────────────────────────────────────────┘
```

| ビュー名 | グルーピング軸                           | T-BOM相当       |
| -------- | ---------------------------------------- | --------------- |
| フォルダ | 従来のフォルダ階層互換                   | 101a ツリー表示 |
| 機器別   | リストタイプ → リスト単位                | 101b 機器別表示 |
| エリア別 | 工区（タグ）→ スキッドGr → 機器          | **新規**        |
| 工種別   | 機器種別（タグ）→ サブ分類 → 機器        | **新規**        |
| 進捗別   | ステータス（積算済/未積算/確定等）       | **新規**        |
| カスタム | ユーザー定義の属性軸で自由にグルーピング | **新規**        |

**設計ポイント:**

- 各ビュータブはドラッグで並び替え可能。よく使うビューを左に配置
- 「カスタムビュー」は属性（タグ）を軸にして自由にネスト可能（例: エリア → 圧力等級 → 機器）
- ビュー設定はユーザーごとに保存され、チーム内で共有もできる

#### フィルターバー

属性タグによるリアルタイムフィルタリング。複数条件のAND/ORをサポート。

```text
フィルター例:
  [工区: A-100, B-200] AND [リストタイプ: 計装弁] AND [メーカ: 横河電機]
```

- フィルター条件はURLパラメータとして保存され、**リンクとして他ユーザーに共有可能**
- 保存済みフィルターのプリセット機能あり（例:「エリアA 回転機のみ」「未積算の計装弁」）

#### 属性タグカラム（Tags列）— N-BOMの核心

各機器に付与されたタグを**バッジ表示**する。「フォルダ型 → 属性型」パラダイムシフトの中核UIパーツ。

```text
タグ表示例:
  [📍 A-100] [🏭 国内調達] [⚡ スキッドGr-01] [≡ 150LB]
```

| タグカテゴリ         | バッジ色  | 機器での使用例       |
| -------------------- | --------- | -------------------- |
| 工区 (Area)          | 🔵 Blue   | A-100, B-200         |
| スキッドGr           | 🟢 Green  | Gr-01, Gr-02         |
| 圧力等級 (Rating)    | 🟠 Orange | 150LB, 300LB         |
| 調達先 (Procurement) | 🟣 Purple | 国内, 海外-中国      |
| 機器種別 (Type)      | 🔴 Red    | 回転機, 静機器, 計装 |
| カスタム             | ⚪ Gray   | ユーザー定義         |

- タグは色分けカテゴリで視覚的に区別
- セル内でタグをクリックすると、同一タグのアイテムをハイライト
- タグの追加・削除はインライン操作（タグセルをダブルクリック → ポップオーバー）

#### インライン編集

T-BOMの「セルの一括編集」機能を、Excelライクなインライン編集に進化させる。

| 操作                    | 挙動                                                         |
| ----------------------- | ------------------------------------------------------------ |
| ダブルクリック          | セル単体の編集モード                                         |
| 複数行選択 + 右クリック | 一括編集メニュー（分番/スキッド番号/機器番号/数量/調達先等） |
| `Ctrl+C` / `Ctrl+V`     | セル値のコピー＆ペースト                                     |
| `Ctrl+Z`                | アンドゥ（操作履歴を保持）                                   |
| ドラッグ                | 行の並べ替え（原価項目の並び順変更）                         |

### 3.5 E. サマリーバー

**目的**: 選択した機器のリアルタイム集計。現行T-BOMでは「原価集計」画面（801）への遷移が必要だった処理を、メイン画面上で即座に確認可能にする。

```text
┌───────────────────────────────────────────────────────────────────────┐
│ 選択中: 計装弁 > 調節弁（3件）                                       │
│ 合計金額: ¥2,060,000  │  合計数量: 6  │  平均単価: ¥383,333          │
│ 合計重量: 1,520 kg    │  合計体積: 12.3 m³                            │
└───────────────────────────────────────────────────────────────────────┘
```

- 選択した行（複数可）の集計値をリアルタイム更新
- クリックで集計項目のカスタマイズが可能（金額/数量/重量/体積などの表示切替）
- フォルダ（グループ）を選択すると配下全機器の集計を表示

## 4. メニュー・ツールバー体系の再設計

### 4.1 現行の問題

T-BOMのメニュー体系の問題点を機器積算の視点で整理する。

| 問題               | 具体例                                                         |
| ------------------ | -------------------------------------------------------------- |
| 同一機能の重複定義 | 「仕様選定」がメニュー/ツールバー/右クリックの3箇所以上に存在  |
| 表示モード依存     | ツリー表示/機器別表示で有効/無効が変わる機能の識別が困難       |
| 認知負荷           | 「原価積算」メニュー配下だけで13項目。機器に関係ない機能も混在 |
| 階層が深い         | 「海外積算>掛率設定」など3階層以上のメニュー探索が必要         |

### 4.2 新メニュー体系

機能を**コンテキスト**ごとに3層に分離する。

```text
第1層: コマンドパレット（⌘K）
  → すべての機能に到達可能な統一的エントリーポイント

第2層: コンテキストアクションバー
  → 選択中の機器に応じて表示されるアクション群

第3層: グローバルメニュー
  → 工番レベルの操作、システム設定、外部連携
```

#### 第2層: コンテキストアクションバー

選択されたアイテムの**種別に応じて動的に変化**するツールバー。

**機器（原価項目）を選択中:**

```text
┌─────────────────────────────────────────────────────────────┐
│ [仕様選定] [リスト編集] [コピー] [切り取り] [貼り付け]      │

└─────────────────────────────────────────────────────────────┘
```

**フォルダ / グループを選択中:**

```text
┌─────────────────────────────────────────────────────────────┐
│ [+ 原価項目追加] [+ フォルダ追加] [フォルダ削除]            │
│ [展開] [畳む] [コピー] [貼り付け] [•••]                    │
└─────────────────────────────────────────────────────────────┘
```

**何も選択していない状態（工番レベル）:**

```text

```

### 5.3 タグによるビュー生成の例

ユーザーがカスタムビューを作成する際のフロー:

```text

3. 第2階層のグルーピング軸を選択: [スキッドGr]
4. 第3階層のグルーピング軸を選択: [機器種別]

▾ 📍 A-100
  ▾ ⚡ Gr-01

      圧力調節弁 PV-201
```

## 7. パフォーマンス対策

**解決する課題**: C4（パフォーマンス劣化）

### フロントエンド

| 対策                   | 説明                                                                    |
| ---------------------- | ----------------------------------------------------------------------- |
| 仮想スクロール         | グリッドに数万行の機器データがあっても、表示行のみDOMレンダリング       |
| 遅延読み込み           | ツリーの子ノードは展開操作時に非同期取得                                |
| クライアントキャッシュ | 一度取得した機器データをローカルキャッシュし、再描画時のAPIコールを抑制 |
| Optimistic Update      | 編集操作はUI上即座に反映し、バックグラウンドでサーバー同期              |

### バックエンド

| 対策                           | 説明                                                                           |
| ------------------------------ | ------------------------------------------------------------------------------ |
| テンポラリテーブル廃止         | 属性ベースのクエリはインデックス付きタグテーブルで処理。動的テーブル生成を排除 |
| インクリメンタル集計           | 全件再集計ではなく、変更差分のみを再計算するイベント駆動型集計                 |
| カーソルベースページネーション | 大量の機器データはカーソルベースのページネーションで段階的に取得               |

## 10. まとめ — T-BOM → N-BOM 変更対照表（機器特化）

| 観点           | T-BOM（現行）                      | N-BOM（新設計）                                                     |
| -------------- | ---------------------------------- | ------------------------------------------------------------------- |
| データモデル   | フォルダ階層依存（最大4階層）      | 属性タグベースのフラットモデル（階層制限なし）                      |
| 表示モード     | ツリー/機器別の2択（画面切替）     | 多次元ビュー（フォルダ/機器別/エリア/工種/進捗/カスタム）をタブ切替 |
| メニュー体系   | 8メニュー×80機能超の固定構造       | コマンドパレット＋コンテキストアクションの動的構造                  |
| 編集方式       | 別画面遷移＋専用ダイアログ         | インライン編集＋ディテールパネル（遷移最小化）                      |
| 集計           | 別画面（原価集計801）で確認        | サマリーバーでリアルタイム表示                                      |
| 過去実績参照   | 不可（PDFが散在）                  | ディテールパネル「実績」タブで類似機器を自動検索                    |
| ブロック管理   | ツールバーから各ブロック画面に遷移 | ディテール「ブロック」タブで所属確認＋編集導線                      |
| スキッドGr     | 専用ダイアログ                     | タグシステムに統合（スキッドGrタグの付与）                          |
| パフォーマンス | テンポラリテーブルによる対症療法   | 仮想スクロール＋インクリメンタル集計                                |
| キーボード操作 | 限定的                             | フルショートカット対応＋コマンドパレット                            |
| カスタマイズ   | なし（全ユーザー共通UI）           | 列設定/ビュー/フィルターをユーザーごとに保存・共有                  |
