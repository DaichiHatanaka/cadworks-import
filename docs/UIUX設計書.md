## 1. 画面概要

### 1.1 目的

本画面は、CADWorx（設計CAD）の機器データとT-BOM（原価管理）の原価項目を手動で紐付けるための中核画面である。自動紐付けで対応できなかった未紐付けデータを作業者が視覚的に確認しながら、手動で紐付け・追加・解除の操作を行う。

### 1.2 コア機能の本質

本画面の本質的な機能は、CADWorx側とT-BOM側のそれぞれから**同じ5つの属性**を取得し、1対1で突き合わせて差異を確認することである。

**比較対象の5属性：**

| No. | 属性名       | CAD側（CWX_DATA） | T-BOM側（TBOM_DATA） | 比較の意味                                     |
| --- | ------------ | ----------------- | -------------------- | ---------------------------------------------- |
| 1   | リストタイプ | LIST_TYPE         | LIST_TYPE            | 機器種別の分類。同一タイプ同士でのみ紐付け可能 |
| 2   | 機器番号     | KIKI_NO           | KIKI_NO              | 機器を特定するTagNo。紐付けの主キー            |
| 3   | 機器名称     | KIKI_BAME         | KIKI_BAME            | 機器の名称。差異があれば設計側で確認が必要     |
| 4   | 数量         | QTY_ORD           | QTY_ORD              | 機器の台数。不一致は原価に直結する重要差異     |
| 5   | 概略仕様     | SHORT_SPEC        | SHORT_SPEC           | 機器の仕様情報。不一致は設計判断が必要         |

**処理の流れ：**

```
┌──────────────┐                              ┌──────────────┐
│  CADWorx     │   5属性を取得                  │  T-BOM       │
│  (設計CAD)   │──────────────┐  ┌──────────────│  (原価管理)  │
│              │              ▼  ▼              │              │
│ ・リストタイプ │        ┌───────────┐          │ ・リストタイプ │
│ ・機器番号    │        │  突き合わせ │          │ ・機器番号    │
│ ・機器名称    │        │  (1対1比較) │          │ ・機器名称    │
│ ・数量       │        └─────┬─────┘          │ ・数量       │
│ ・概略仕様    │              │                │ ・概略仕様    │
└──────────────┘              ▼                └──────────────┘
                       ┌───────────┐
                       │ 差異の確認 │
                       │ ・一致     │
                       │ ・不一致   │
                       │ ・片方のみ │
                       └───────────┘
```

本画面のフィルタ機能（リストタイプ、ステータス）、保存・破棄、Excel出力等はすべて、この「5属性を比較して対応づける」作業を効率的・安全に実行するための**補助機能**として位置づけられる。

### 1.3 表示条件（画面遷移元）

「CADWorx P&ID取込」画面（TBOM-610）にて「開始」ボタンを押し、遷移先のダイアログで「紐付け開始」ボタンを押した際に本画面へ遷移する。

### 1.4 画面タイトルの動的変更

画面タイトルは、前画面（TBOM-610）の「取込対象」チェックの状態により動的に変化する。取込対象の選択内容に応じて、対象リストタイプの範囲が画面タイトルに反映される。

## 2. UI設計思想

### 2.1 デザイン原則

コア機能「5属性の比較と対応づけ」から逆算し、以下の4原則でUI全体を設計する。

| 原則                                       | 説明                                                                                                                    | 根拠                                                                                                              |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **① 対称性（ミラーレイアウト）**           | CAD側とT-BOM側を常に左右対称に配置する。同じ5属性を同じカラム順で並べ、「同じものを比べている」ことを視覚的に自明にする | コア機能が「同じ5属性の突合」である以上、UIも左右対称が自然。作業者は視線を横にスライドするだけで差異を把握できる |
| **② 差異の即時可視化**                     | 値が異なるセルをオレンジ色で自動ハイライトし、作業者が「どこを見ればいいか」を一瞬で把握できるようにする                | 数百件のレコードから差異を目視で探すのは非現実的。システムが差異を可視化し、作業者は判断に集中できるようにする    |
| **③ 段階的開示（Progressive Disclosure）** | 作業の3段階（①未紐付けを紐付ける → ②紐付け結果を確認する → ③差異を詳細に見る）に応じて情報量を段階的に増やす            | 全情報を一度に見せると認知負荷が高い。作業フェーズに合わせて必要な情報だけを提示する                              |
| **④ 進捗の常時フィードバック**             | 「あと何件残っているか」「全体の何%が完了しているか」を常に表示し、作業のゴールを明確にする                             | 紐付け作業は機器数が多いほど長時間化する。進捗が見えないと作業者の集中力とモチベーションが低下する                |

### 2.2 ユーザーのメンタルモデル

作業者の思考プロセスに沿ったUI構成とする。

```
作業者の思考フロー：

  ❶「まだ紐付いていないCAD機器はどれか？」
      → 上段左ペイン（CAD未紐付け一覧）を見る

  ❷「この機器に対応するT-BOM項目はどれか？」
      → 上段右ペイン（T-BOM未紐付け一覧）を見る
      → 同一リストタイプに自動フィルタされている

  ❸「この2つを紐付けてよいか？」
      → 5属性のミラー比較で差異を確認 → 紐付け実行

  ❹「紐付け結果は正しいか？差異はないか？」
      → 下段（紐付け済み一覧）で確認
      → 差異セルがオレンジでハイライトされている

  ❺「全部終わったか？」
      → プログレスバーとカウンターで確認
```

### 3.2 レイアウト設計の根拠

| 設計判断                           | 根拠（コア機能との関係）                                                                                          |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| 上段を左右2ペインに分割            | 「CADの5属性」と「T-BOMの5属性」を左右対称に並べることで、比較対象が同じ構造であることが一目でわかる              |
| 中央に操作ボタンを縦配置           | 左（CAD）→ 右（T-BOM）の方向に「紐付け」、右 → 左の方向に「解除」。データの流れとボタンの位置が一致する           |
| 下段のミラーテーブル               | 紐付け済みデータの5属性を左右対称に表示。同じ行の同じカラム位置を見るだけでCAD vs T-BOMの差異がわかる             |
| 差異セルのオレンジハイライト       | 5属性の比較結果を色で即座にフィードバック。作業者は「オレンジのセルだけ見ればよい」という明快なルールで作業できる |
| プログレスバーをヘッダーに常時表示 | 「残り何件か」が常に見えることで、作業の完了見込みが立ちやすくなる                                                |
| フッターにアクションバーを集約     | 保存・破棄・終了は作業の「区切り」に使うもの。画面最下部に配置することで、作業中の誤操作を防ぐ                    |

## 4. インタラクションデザイン

### 4.1 紐付け操作の3つの方法

コア操作である「紐付け」を、作業者の習熟度に応じて3段階の操作方法で提供する。

| 操作方法               | 手順                                                                      | 対象ユーザー              |
| ---------------------- | ------------------------------------------------------------------------- | ------------------------- |
| **ボタン操作**（基本） | ①上段左でCAD行を選択 → ②上段右でT-BOM行を選択 → ③中央「紐付け」ボタン押下 | 初回利用者・正確性重視    |
| **ドラッグ＆ドロップ** | 上段左のCAD行を上段右のT-BOM行へドラッグ（またはその逆）                  | 習熟した作業者・効率重視  |
| **ダブルクリック追加** | 上段左のCAD行をダブルクリック → T-BOM紐付けなしで即座に「追加」           | CADのみの機器を素早く処理 |

### 4.2 連動フィルタの動作

「T-BOM側をCAD選択行のリストタイプで自動フィルタ」チェックボックスの動作を定義する。この機能は**コア機能（同一リストタイプ同士の比較）を直接支援する最重要の補助機能**である。

| 状態                     | 動作                                                                          | UX上の意図                                                                         |
| ------------------------ | ----------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **ON（デフォルト推奨）** | 上段左でCAD行を選択すると、上段右が自動的にそのリストタイプのみに絞り込まれる | 紐付け可能な候補だけが表示されるため、作業者は「選ぶだけ」で済む。認知負荷を最小化 |
| OFF                      | 上段右は全T-BOM未紐付けデータを表示                                           | 異なるリストタイプの全体状況を俯瞰したい場合に使用                                 |

**デフォルトONの理由：** 紐付けルール上、同一リストタイプ同士でしかマッチングできない。したがって、異なるリストタイプの項目を表示する意味は通常ない。ONをデフォルトとすることで、作業者が毎回フィルタをかける手間を省く。

### 4.3 選択行プレビュー（インライン展開）

上段左右のテーブルで行を選択すると、テーブル内にインライン展開で詳細情報を表示する。旧画面の「表⑤ 選択行詳細」を独立エリアではなくインライン化することで、視線移動を削減する。

```
上段左テーブル内でのインライン展開イメージ：

  │ L002 │ P-101 │ ポンプA  │ 2 │ 横型遠心... │  ← 選択行
  │ ┌─────────────────────────────────────┐ │  ← 展開部分
  │ │ 機器番号: P-101                      │ │
  │ │ 機器名称: ポンプA                    │ │
  │ │ 数量:     2                         │ │
  │ │ 概略仕様: 横型遠心ポンプ 50A 0.75kW  │ │
  │ └─────────────────────────────────────┘ │
  │ L002 │ P-102 │ ポンプB  │ 1 │ 立型渦巻... │
```

### 4.4 差異ハイライトの視覚ルール

下段の紐付け済みミラーテーブルにおける、差異の可視化ルールを定義する。

| 状態            | 表示                     | 色               | 意味                                       |
| --------------- | ------------------------ | ---------------- | ------------------------------------------ |
| 完全一致        | 通常表示                 | 白背景           | CADとT-BOMの当該属性が同一値               |
| **値不一致**    | **オレンジハイライト**   | **#FFA500 背景** | **CADとT-BOMで値が異なる。設計確認が必要** |
| CADのみ（追加） | T-BOM側グレーアウト      | #E0E0E0 背景     | T-BOM側に対応項目なし。新規追加扱い        |
| 未保存          | ステータス列にバッジ表示 | 黄色バッジ       | まだ保存されていない変更あり               |
| 保存済み        | ステータス列にバッジ表示 | 緑バッジ         | 確定済み                                   |

**差異ハイライトの対象属性：** 5属性のうち、以下を個別セル単位で比較しハイライトする。

| 属性         | 比較対象           | 不一致時の影響度                                               |
| ------------ | ------------------ | -------------------------------------------------------------- |
| リストタイプ | LIST_TYPE同士      | 紐付けルール上、不一致は発生しない（同一タイプのみ紐付け可能） |
| 機器番号     | KIKI_NO同士        | 中：番号違いは確認必要                                         |
| 機器名称     | KIKI_BAME同士      | 低〜中：表記揺れの可能性あり                                   |
| **数量**     | **QTY_ORD同士**    | **高：原価に直結。最優先で確認**                               |
| **概略仕様** | **SHORT_SPEC同士** | **中〜高：仕様違いは設計判断が必要**                           |

## 5. コンポーネント仕様

### 5.1 ヘッダー：プログレスバー

紐付け進捗を常時表示し、作業のゴールを明確化する。

| 項目           | 仕様                                                           |
| -------------- | -------------------------------------------------------------- |
| 表示形式       | 水平プログレスバー ＋ テキスト（「紐付け進捗 78% (156/200)」） |
| 分母           | CADデータ総件数（自動紐付け済み＋手動紐付け済み＋未紐付け）    |
| 分子           | 紐付け済み件数（自動＋手動。ステータスに関わらずカウント）     |
| 色             | 0-49%: 赤系、50-89%: 黄系、90-100%: 緑系                       |
| 更新タイミング | 紐付け・追加・解除操作のたびにリアルタイム更新                 |

| 機器名称     | CWX_DATA   | KIKI_BAME    | 文字列   | 機器の名称                         |

## 9. 自動紐付けロジック

### 9.1 概要

本画面の表示前に、システムは自動紐付け処理を実行する。以下のルールに従い、CAD側とT-BOM側のレコードを自動的にマッチングする。自動で紐付かなかったレコードが上段の左右テーブルに残り、作業者の手動操作対象となる。

### 9.2 マッチングアルゴリズム

**ステップ1：CAD側レコードを起点とした巡回**

CAD側のレコードから順に巡り、LIST_TYPEが一致しているT-BOM側のレコードの中から対応先を探す。

**ステップ2：紐付け判定条件**

LIST_TYPEとKIDが双方一致しているレコードに対して、以下の条件で紐付けの有効性と状態を判定する。

| 条件                                              | 判定結果   | ステータス           | 動作                                             |
| ------------------------------------------------- | ---------- | -------------------- | ------------------------------------------------ |
| LIST_TYPE一致 ＋ KID一致 ＋ ID_COUNT完全一致      | 紐付け有効 | 変更なし（保存済み） | 新たな原価項目を追加しない                       |
| LIST_TYPE一致 ＋ KID一致 ＋ ID_COUNT下5桁のみ一致 | 紐付け有効 | **未保存**           | 前回の紐付け先とは異なるため、作業者の確認が必要 |
| LIST_TYPE一致 ＋ KID不一致                        | 紐付け不可 | —                    | 未紐付けとして上段左右に残留                     |
| LIST_TYPE不一致                                   | 紐付け不可 | —                    | 未紐付けとして上段左右に残留                     |

### 9.3 マッチング例

**入力データ：**

|     | CAD側     |     |          |     | T-BOM側   |     |          |
| --- | --------- | --- | -------- | --- | --------- | --- | -------- |
| No. | LIST_TYPE | KID | ID_COUNT | No. | LIST_TYPE | KID | ID_COUNT |
| ①   | L002      | -1  | 1000000  | ①   | L002      | 1   | 1000001  |
| ②   | L002      | 1   | 0000001  | ②   | L002      | 2   | 1000002  |
| ③   | L002      | 1   | 2000001  | ③   | L002      | 3   | 2000003  |
| ④   | L002      | 2   | 1000002  | ④   | L003      | 4   | 4000001  |

**マッチング結果（下段ミラーテーブルに表示）：**

| CAD側     |     |          | →   | T-BOM側   |     |          | ステータス                             |
| --------- | --- | -------- | --- | --------- | --- | -------- | -------------------------------------- |
| LIST_TYPE | KID | ID_COUNT |     | LIST_TYPE | KID | ID_COUNT |                                        |
| L002      | -1  | 1000000  |     | —         | —   | —        | 追加（T-BOM対応なし）                  |
| L002      | 1   | 0000001  |     | L002      | 1   | 1000001  | **未保存**（ID_COUNT下5桁一致: 00001） |
| L002      | 1   | 2000001  |     | —         | —   | —        | 追加（T-BOM対応なし）                  |
| L002      | 2   | 1000002  |     | L002      | 2   | 1000002  | 保存済み（ID_COUNT完全一致）           |

**判定理由：**

- CAD①（KID=-1）：T-BOM側にKID=-1が存在しないため紐付け不可 → 「追加」扱い
- CAD②（KID=1, ID_COUNT=0000001）：T-BOM①（KID=1, ID_COUNT=1000001）とKID一致。ID_COUNT下5桁「00001」が一致 → 紐付け有効だが前回とは異なるため「未保存」
- CAD③（KID=1, ID_COUNT=2000001）：T-BOM側にKID=1かつ未使用のレコードなし → 「追加」扱い
- CAD④（KID=2, ID_COUNT=1000002）：T-BOM②（KID=2, ID_COUNT=1000002）とKID・ID_COUNT完全一致 → 「変更なし」保存済み

## 11. 処理シーケンス

### 11.1 画面表示時

```
1. TBOM-610から取込対象情報（工番、ケース、施工区分、対象リストタイプ）を受け取る
2. CADデータ（CWX_DATA）を読み込む
3. T-BOMデータ（TBOM_DATA）を読み込む
4. 自動紐付けロジック実行（セクション9参照）
5. 結果を振り分けて表示：
   ├→ 紐付け成功 → 下段ミラーテーブル（ステータス：保存済み or 未保存）
   ├→ 紐付け不可（CAD側）→ 上段左テーブル
   └→ 紐付け不可（T-BOM側）→ 上段右テーブル
6. プログレスバーを初期表示（紐付け済み件数 / 全CAD件数）
7. 対象工番をロック（排他制御開始）
```

### 11.2 手動紐付け操作

```
1. 作業者が上段左のCADデータを選択
2. 入力②がONの場合、上段右が自動でリストタイプフィルタされる
3. 作業者が上段右のT-BOMデータを選択（または「追加」を選択）
4. 「紐付け」or「追加」ボタン押下
5. バリデーション（リストタイプチェック等）
6. 下段ミラーテーブルに追加（ステータス：「未保存」）
7. 上段左右から該当行を削除
8. プログレスバー更新
9. 5属性の差異チェック → 差異セルにオレンジハイライト適用
```

### 11.3 保存操作

```
1. 「保存」ボタン押下
2. 下段テーブルの「未保存」レコードをDBに反映
3. ステータスバッジを黄→緑に更新
4. トースト通知「XX件の紐付けを保存しました。」
```

| 詳細表示   | 独立エリア（表⑤）をトグル表示 | インライン展開（行内に詳細を折り畳み表示）   | 視線移動を削減。コンテキストを維持したまま詳細確認           |
| フィルタ   | ドロップダウン選択のみ        | 検索付きドロップダウン＋テキストフリーサーチ | 大量データ時の探索効率向上                                   |
| 進捗把握   | 件数を自分で数える            | プログレスバー＋残件数を常時表示             | 「あとどれくらいか」の即時フィードバック                     |
| 保存方式   | ローカル保存                  | サーバー保存＋自動バックアップ＋排他制御     | データ消失防止、同時編集防止                                 |
| Excel出力  | ローカルファイル出力          | サーバー側で生成してダウンロード             | 環境依存の排除                                               |

### 13.2 排他制御

| 要件       | 説明                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 工番ロック | 本画面を開いた時点で対象工番をロックし、他ユーザーの編集を防止する   |
| ロック表示 | 他ユーザーがロック中の場合、ロック者と開始時刻を画面上に表示する     |
| 自動解放   | 画面を閉じた際、またはセッションタイムアウト時にロックを自動解放する |

### 13.3 レスポンシブ対応方針

| 画面幅      | レイアウト                                                             |
| ----------- | ---------------------------------------------------------------------- |
| 1440px以上  | 標準レイアウト（上段2ペイン＋下段フルワイド）                          |
| 1024-1439px | 上段2ペイン幅を縮小、概略仕様カラムをツールチップ化                    |
| 1024px未満  | 本画面は精密な比較作業のため、最低幅1024pxを要求。それ未満では警告表示 |
