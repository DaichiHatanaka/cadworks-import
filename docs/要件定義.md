## 1. 目的

本書は、`docs/UIUX設計書.md` に記載されたスコープのみをMVPとして実装するための要件を定義する。  
対象は **CADWorx手動紐付け画面（TBOM-611）** のみとし、周辺業務機能は本MVPの対象外とする。

---

## 2. MVPスコープ

### 2.1 対象

- TBOM-611画面の表示・操作・保存
- CADデータとT-BOMデータの手動紐付け
- 5属性のミラー比較と差異可視化
- 自動紐付け（画面表示前処理）
- 排他制御（工番ロック）

### 2.2 比較対象の5属性（必須）

| No. | 属性名       | CAD側      | T-BOM側    |
| --- | ------------ | ---------- | ---------- |
| 1   | リストタイプ | LIST_TYPE  | LIST_TYPE  |
| 2   | 機器番号     | KIKI_NO    | KIKI_NO    |
| 3   | 機器名称     | KIKI_BAME  | KIKI_BAME  |
| 4   | 数量         | QTY_ORD    | QTY_ORD    |
| 5   | 概略仕様     | SHORT_SPEC | SHORT_SPEC |

## 3. 機能要件（MVP）

### 3.1 画面遷移・初期表示

| 要件ID     | 要件名           | 内容                                                                              |
| ---------- | ---------------- | --------------------------------------------------------------------------------- |
| FR-MVP-001 | 画面遷移         | TBOM-610から「紐付け開始」でTBOM-611へ遷移できること                              |
| FR-MVP-002 | 初期読込         | 工番・ケース・施工区分・対象リストタイプを引き継ぎ、CAD/T-BOMデータを読み込むこと |
| FR-MVP-003 | 自動紐付け前処理 | 画面表示前に自動紐付けロジックを実行し、結果を上段/下段へ振り分けること           |
| FR-MVP-004 | 進捗表示         | ヘッダーに紐付け進捗（件数・率）を表示し、操作のたび更新すること                  |

### 3.2 未紐付け作業エリア（上段）

| 要件ID     | 要件名          | 内容                                                                           |
| ---------- | --------------- | ------------------------------------------------------------------------------ |
| FR-MVP-010 | 上段2ペイン表示 | CAD未紐付け一覧（左）とT-BOM未紐付け一覧（右）を表示すること                   |
| FR-MVP-011 | カラム統一      | 上段左右のカラム順を同一にし、比較しやすくすること                             |
| FR-MVP-012 | 連動フィルタ    | CAD行選択時、T-BOM側を同一リストタイプに自動フィルタできること（デフォルトON） |
| FR-MVP-013 | 行詳細表示      | 選択行の詳細情報をインライン展開で表示できること                               |

### 3.3 紐付け操作

| 要件ID     | 要件名         | 内容                                                                 |
| ---------- | -------------- | -------------------------------------------------------------------- |
| FR-MVP-020 | 紐付け         | CAD選択行とT-BOM選択行を紐付けし、下段へ移動できること               |
| FR-MVP-021 | 追加           | CAD選択行をT-BOM対応なしで下段へ追加できること                       |
| FR-MVP-022 | 紐付け解除     | 下段選択行を解除し、上段へ戻せること                                 |
| FR-MVP-023 | 複数操作方式   | ボタン操作に加え、ドラッグ&ドロップ/ダブルクリック追加を提供すること |
| FR-MVP-024 | バリデーション | 異なるリストタイプ同士の紐付けを禁止し、警告表示すること             |

### 3.4 紐付け済み比較エリア（下段）

| 要件ID     | 要件名         | 内容                                                                      |
| ---------- | -------------- | ------------------------------------------------------------------------- |
| FR-MVP-030 | ミラーテーブル | CAD側5属性とT-BOM側5属性を左右対称で表示すること                          |
| FR-MVP-031 | 差異ハイライト | 不一致セルをオレンジ表示（#FFA500）すること                               |
| FR-MVP-032 | 追加表示       | T-BOM対応なし行はT-BOM側をグレーアウト表示すること                        |
| FR-MVP-033 | ステータス表示 | 行ステータス（未保存/保存済み）をバッジ表示すること                       |
| FR-MVP-034 | 絞り込み検索   | リストタイプ・ステータス・テキスト検索（機器番号/機器名称）を提供すること |

### 3.5 保存・終了

| 要件ID     | 要件名    | 内容                                                                     |
| ---------- | --------- | ------------------------------------------------------------------------ |
| FR-MVP-040 | 保存      | 未保存行を確定保存し、保存済みへ更新すること                             |
| FR-MVP-041 | 破棄      | 未保存変更を前回保存状態へ戻せること                                     |
| FR-MVP-042 | 終了確認  | 未保存データ有無に応じた確認ダイアログを表示し、終了可否を判断できること |
| FR-MVP-043 | Excel出力 | 下段テーブル内容をExcel出力できること                                    |

### 3.6 自動紐付けロジック

| 要件ID     | 要件名         | 内容                                                          |
| ---------- | -------------- | ------------------------------------------------------------- |
| FR-MVP-050 | マッチング条件 | LIST_TYPE一致を前提にKID/ID_COUNT条件で自動紐付け判定すること |
| FR-MVP-051 | 状態判定       | ID_COUNT完全一致は保存済み、下5桁一致は未保存として扱うこと   |
| FR-MVP-052 | 未紐付け残留   | 自動紐付け不可レコードは上段の未紐付け一覧に残すこと          |

## 4. 非機能要件（MVP最小）

| 要件ID      | 要件名       | 内容                                                       |
| ----------- | ------------ | ---------------------------------------------------------- |
| NFR-MVP-001 | 排他制御     | 画面表示時に対象工番をロックし、同時編集を防止すること     |
| NFR-MVP-002 | ロック解放   | 画面終了またはタイムアウト時にロックを解放すること         |
| NFR-MVP-003 | データ整合性 | 紐付けは同一LIST_TYPE間のみ許可し、不正更新を防止すること  |
| NFR-MVP-004 | 操作応答性   | フィルタ・選択・ハイライト反映が実用上遅延なく動作すること |

## 5. 画面・データ要件（MVP）

### 5.1 対象画面

- TBOM-611：CADWorx手動紐付け画面

### 5.2 使用テーブル

| テーブル名 | 用途                                    |
| ---------- | --------------------------------------- |
| CWX_DATA   | CAD未紐付け/紐付け済み（CAD側）表示     |
| TBOM_DATA  | T-BOM未紐付け/紐付け済み（T-BOM側）表示 |

### 5.3 主な管理キー

| 項目           | 役割                          |
| -------------- | ----------------------------- |
| LIST_TYPE      | 同一タイプ間でのみ紐付け可能  |
| KID            | 自動紐付けの主要キー          |
| ID_COUNT       | 変更なし/未保存の状態判定キー |
| CWX_LINKED_FLG | 紐付け状態の管理              |

以下は `UIUX設計書.md` の当該スコープ外のため、MVP対象外とする。

- mdb書き戻し出力、T-BOM全体更新エンジンなど周辺連携の拡張機能
- ロール管理、監査ログ、詳細性能要件など運用拡張項目

---

## 7. 受け入れ基準（MVP）

- TBOM-611で、上段未紐付けから下段紐付け済みへの操作（紐付け/追加/解除）が実行できる
- 下段で5属性を左右比較でき、不一致セルがオレンジで可視化される
- 保存/破棄/終了の状態遷移が仕様通りに動作する
- 連動フィルタ・検索で対象レコードを絞り込める
- 画面ロックにより同一工番の同時編集が抑止される
