# N-BOM 機器ブロックマスター管理システム 要件定義書

**文書番号**: NBOM-REQ-001  
**版数**: 4.0（v2 UI/UX設計 + v3 データモデルレビュー統合版）  
**作成日**: 2026-03-10  
**ステータス**: ドラフト  
**スコープ**: MVP（Phase 1）

---

## 1. 概要

### 1.1 システムの目的

水処理プラント設計において、標準化された機器ブロックのマスターデータを一元管理する新BOMシステム「**N-BOM**」を構築する。従来Excelベースで管理されていた機器ブロックデータ一覧表の情報をデータベース化し、ブロックの登録・検索・編集・版管理を効率化する。将来的にはプロジェクト（案件）固有のカスタマイズ管理まで対応する。

### 1.2 背景と課題

現行運用では、機器ブロックデータ一覧表（`.xls`形式）とブロックフロー図（PDF）がファイルベースで管理されている。この運用では以下の課題がある。

ファイルの肥大化と検索性の低下が進んでおり、1ブロックあたり数百行×192列のデータが存在する。タンクテーブル（設計条件に応じた仕様バリエーション）が塔径×高さの組み合わせで膨大な行数に展開されるため、全体像の把握が困難になっている。複数担当者による同時編集ができず、版管理も手動で行っている状況である。さらに、Excelの表形式ではBOMの親子関係や機器間の接続関係が直感的に把握できず、ブロックフロー図（PDF）と行き来しながら確認する必要がある。

加えて、D01〜D50列の意味がリストタイプごとに異なるにもかかわらずメタ定義が存在しないため、「列番号を暗記している人しか使えない」という属人化の問題がある。標準ブロックを案件に適用した際のカスタマイズ（メーカー変更等）を管理する仕組みも存在しない。

### 1.3 N-BOM階層構造

N-BOMは装置を構成する購入品・外注品を4階層で定義する。

**第一階層（機能ブロック）** → **第二階層（単位ブロック）** → **第三階層（機器ブロック）** → **第四階層（原価項目）**

本システムが管理する「機器ブロック」は第三階層に位置し、ユニットを構成するサブユニットレベルの単位である。具体的にはポンプ、バルブ、計器、製缶品（タンク・塔類）などの機器群を一つのまとまりとしてブロック化したものを指す。

### 1.4 アーキテクチャ方針: Master-Instance パターン

N-BOMのデータアーキテクチャは**Master-Instance分離**を基本思想とする。

```
標準マスタ層（Standard）          案件層（Project）← Phase 2
┌─────────────────┐          ┌──────────────────┐
│ block_master     │───引用──→│ project_block     │
│  └ cost_item     │          │  └ pj_item_override│
│     └ item_spec  │          │                    │
└─────────────────┘          └──────────────────┘

マスタは「変えない」             案件固有の差分だけ保持
全案件の共通基盤               上書き（Override）方式
```

MVPではマスタ層を実装し、案件層はテーブル定義のみ用意してPhase 2で実装する。

### 1.5 MVPスコープ

**Phase 1（MVP）含む**: マスタ層テーブル全量、パラメータ定義マスタ、Excelインポート、ブロックワークスペース（3ペインUI）、マトリクスビュー、PDF表示、基本認証

**Phase 2で追加**: 案件層（project / project_block / pj_item_override）、PDFホットスポット双方向連動、図面注釈エディタ、ユーザー権限管理、3Dビューア統合

## 2. UI/UXデザイン方針

### 2.1 設計原則

**原則1: コンテキスト同時表示（Split-View First）**
エンジニアが「図面を見ながらBOMを確認する」という自然なワークフローをそのままデジタル化する。階層ツリー・図面PDF・詳細データの3つの情報を常に同一画面内で参照できる3ペインレイアウトを基本とする。

**原則2: 階層の可視性（Hierarchy Always Visible）**
BOM構造は常にナビゲーション可能な状態で表示する。現在どの階層のどの要素を見ているかを、パンくずリスト・ツリーハイライト・図面上のハイライトの三重の手段で明示する。

**原則3: 図面とデータの双方向連動（Bi-directional Linking）**
図面PDF上の機器シンボルをクリックすると対応するBOMデータが選択され、逆にBOMツリーの項目を選択すると図面上の対応箇所がハイライトされる。図面は「ただの参考資料」ではなく、ナビゲーションの一部として機能する。（双方向連動の本格実装はPhase 2）

**原則4: 段階的詳細開示（Progressive Disclosure）**
初期表示ではブロックの俯瞰（構成機器数・主機器情報・ステータス）を示し、ユーザーの操作に応じてタンクテーブル・設計条件・電動機仕様などの詳細情報を段階的に表示する。192列のデータを一度に見せない。

**原則5: 設計条件による文脈切替（Context Switching by Design Condition）**
バリエーション刻み（例: φ1200×H3500）を選択すると、画面全体のコンテキストがその設計条件に切り替わり、全原価項目の仕様・図面・配管パターンが連動して更新される。

**原則6: 自己記述型UI（Self-Describing UI）**
D01〜D50列の意味は`param_def`マスタに定義し、UIはリストタイプに応じたラベル・入力種別・選択肢を動的に生成する。「D02は口径」ではなく「口径: 65A」と表示する。

### 2.2 メインレイアウト: 3ペイン構成

```
┌──────────────────────────────────────────────────────────────────┐
│  グローバルヘッダー（ブロック検索 / パンくず / 設計条件セレクタ）   │
├────────────┬──────────────────────┬─────────────────────────────┤
│            │                      │                             │
│  左ペイン    │  中央ペイン            │  右ペイン                    │
│  BOM構造    │  図面ビューア          │  詳細プロパティ               │
│  ツリー     │  (PDF / Schematic)   │  パネル                     │
│            │                      │                             │
│  ・階層ツリー │  ・PDF表示/拡縮       │  ・選択項目の設計条件          │
│  ・リスト切替 │  ・機器ハイライト      │  ・バリエーション仕様         │
│  ・フィルタ  │  ・アノテーション      │  ・メーカー/型式             │
│  ・ドラッグ  │  ・比較表示           │  ・図面リンク                │
│   並替     │                      │  ・電動機仕様                │
│            │                      │  ・配管パターン              │
├────────────┴──────────────────────┴─────────────────────────────┤
│  フッターバー（ステータス / 統計 / アクション）                     │
└──────────────────────────────────────────────────────────────────┘
```

**左ペイン（BOM構造ツリー）**: 幅280px〜360px、リサイズ可能。ブロック内の全原価項目を階層構造またはリストタイプ別グループで表示する。

**中央ペイン（図面ビューア）**: 可変幅、最大化可能。ブロックフローPDFをインライン表示する。MVP段階ではPDF表示＋手動ハイライト、Phase 2で双方向連動を実装する。

**右ペイン（詳細プロパティ）**: 幅320px〜480px、リサイズ可能。左ペインまたは中央ペインで選択した機器の詳細情報をタブ形式で表示する。入力フォームは`param_def`マスタに基づいて動的に生成する。

### 2.3 BOM構造の表示パターン

#### 2.3.1 ツリービュー（デフォルト）

N-BOMの親子関係をツリー形式で表示する。

```
▼ E000000028  重力式二層ろ過器(ポンプ移送)
  ├─ ★ T-GFP01  重力式二層ろ過器 [L201 製缶]     ← 主機器
  ├─ 梯子・手摺・踊場 [L201 製缶]
  ├─ 共通歩廊・階段 [L201 製缶]
  ├─ P-GFP01 ろ過ポンプ [L002 ポンプ]
  ├─ ろ材
  │   ├─ アンスラサイト [L136]
  │   ├─ 砂 [L136]
  │   └─ 砂利1〜5 [L136]
  ├─ ストレーナー
  │   ├─ BFストレーナー [L083]
  │   └─ GFストレーナー [L083]
  ├─ 計装品
  │   ├─ FI-GFP01 逆洗水入口流量計 [L121]
  │   ├─ PI-GFP01 ポンプ出口圧力計 [L122]
  │   ├─ LICA-GFP01 液面計 [L831]
  │   └─ ...
  ├─ 自動弁
  │   ├─ FV-GFP01 原水入口 [L151]
  │   └─ ...
  └─ 手動弁
      ├─ V-GFP01 原水入口サンプリング [L152]
      └─ ...
```

ツリーノードの表示は「機器番号 + 名称 + [リストタイプ]」を基本とし、主機器は★アイコンで強調する。除外フラグがONの項目はグレーアウト＋取消線で表示する。ツリーのグルーピングは`list_type_master.category`（構造物/回転機/内蔵品/計装品/自動弁/手動弁）で自動分類する。

#### 2.3.2 カテゴリービュー

リストタイプ別にカード形式で分類表示する。各カテゴリーは`list_type_master.display_color`に基づくカラーコード付き。

#### 2.3.3 マトリクスビュー（バリエーション俯瞰）

行に原価項目（`cost_item`）、列にバリエーション刻み（`variation_step`）をとったマトリクス表示。セルの値は`item_step_spec`のメーカー型式や`item_param_value`の主要パラメータを表示する。仕様が変わる境界セルをハイライトし、エンジニアが「どの塔径から口径が変わるか」を一目で把握できるようにする。

```
                  φ1200    φ1400    φ1600    φ1800    ...    φ7000
 ───────────────────────────────────────────────────────────────
 ろ過器本体    SS400    SS400    SS400    SS400    ...   SS400
 原水入口弁    65A      80A     [100A]   [100A]   ...   300A
 ろ過ポンプ    酉島     酉島     酉島     酉島     ...   酉島
 配管口径      65A      80A      100A     100A     ...   300A
```

#### 2.3.4 フラットリストビュー

従来のExcel一覧表に近いフラットテーブル表示。列の表示/非表示をカスタマイズ可能。ソート・フィルタ・一括編集に対応する。

## 3. 画面詳細設計

### 3.1 グローバルヘッダー

**パンくずナビゲーション**: `ブロック一覧 > E000000028 重力式二層ろ過器 > FV-GFP01 原水入口自動弁` のように現在のコンテキストを表示する。

**設計条件セレクタ（コンテキストスイッチャー）**: `variation_step`の刻みを選択するドロップダウン。選択すると画面全体のコンテキストが切り替わる。

```
┌─ 設計条件 ─────────────────────────────────┐
│  塔径×高さ: [φ1800 × H3500 ▼]              │
│  配管クラス: [1W1P ▼]                       │
└────────────────────────────────────────────┘
```

このセレクタを変更すると、右ペインの`item_step_spec` + `item_param_value`の値、フッターの`piping_pattern`情報がすべて連動更新される。

**クイック検索**: ブロック番号・名称・機器番号でのインクリメンタルサーチ。`Ctrl+K` で即座にフォーカス。

**アクションメニュー**: Excelインポート、PDF添付、版管理操作をまとめたメニュー。

### 3.2 左ペイン: BOM構造パネル

#### 3.2.1 ビュー切替タブ

| タブ       | 内容               | 主な用途                      |
| ---------- | ------------------ | ----------------------------- |
| ツリー     | 階層ツリー表示     | BOM構成の確認・ナビゲーション |
| カテゴリー | リストタイプ別分類 | 機器種別ごとの一覧確認        |
| リスト     | フラットテーブル   | 一括検索・ソート・フィルタ    |

#### 3.2.2 フィルタバー

**テキスト検索**: 機器番号・名称でのインクリメンタルフィルタ。

**リストタイプフィルタ**: `list_type_master`に基づくチップ形式のトグルボタン。

```
[L201 製缶] [L002 ポンプ] [L151 自動弁] [L152 手動弁] [L831 計装] ...
```

**表示オプション**: 除外フラグ（`is_excluded`）項目の表示/非表示切替。

#### 3.2.3 ツリーノードのインタラクション

**シングルクリック**: 右ペインに詳細プロパティを表示。中央ペインの図面上で対応する機器をハイライト（Phase 2で自動連動化）。

**ダブルクリック**: 右ペインの編集モードに遷移。

**右クリック**: コンテキストメニュー（編集、複製、削除、除外フラグ切替、上下移動）を表示。

**ドラッグ&ドロップ**: `cost_item.sort_order`を変更。

#### 3.2.4 ツリーノードの情報表示

```
┌────────────────────────────────────────┐
│ ★ [L201] T-GFP01                      │
│   重力式二層ろ過器                      │
│   φ1800×H3500  SS400                  │ ← 選択中の設計条件での主要仕様
│   ■■■■■■■□□□  21仕様               │ ← バリエーション充足率バー
└────────────────────────────────────────┘
```

充足率バーは`item_step_spec`の入力済み行数 / `variation_step`の総数で算出する。

#### 3.2.5 サマリーフッター

```
原価項目: 39件（除外3件）| 主機器: 1件 | 自動弁: 7件 | 手動弁: 11件
```

### 3.3 中央ペイン: 図面ビューア

#### 3.3.1 PDF表示（MVP）

`block_document`（`doc_type='block_flow'`）に登録されたPDFをインライン表示する。拡大/縮小、パン操作、ページ切替、回転に対応する。

#### 3.3.2 図面未登録時の代替表示

PDFが未登録の場合、ドラッグ&ドロップでPDFをアップロードできるドロップゾーンを表示する。

#### 3.3.3 機器ホットスポット連動（Phase 2）

Phase 2でPDF上に機器ホットスポットオーバーレイを実装し、BOMツリーとの双方向選択連動を実現する。

### 3.4 右ペイン: 詳細プロパティパネル

#### 3.4.1 タブ構成

| タブ名         | データソース                                        | 表示条件                   |
| -------------- | --------------------------------------------------- | -------------------------- |
| 概要           | `cost_item` + 選択中`item_step_spec`                | 常時                       |
| 設計条件       | `item_param_value`（D列）＋ `param_def`でラベル解決 | 常時                       |
| バリエーション | 全`variation_step`に対する`item_step_spec`一覧      | 刻みがある場合             |
| 電動機         | `motor_spec`                                        | リストタイプがL002等の場合 |
| 図面           | `item_step_spec.std_dwg_no` + `url_3d`              | 図面リンクがある場合       |

#### 3.4.2 設計条件タブの動的フォーム生成

`param_def`マスタに基づき、リストタイプに応じた入力フォームを自動生成する。

```
┌─ 設計条件（L151 自動弁リスト）──────────────┐
│                                              │
│  弁種      [BF          ▼]    ← select     │
│  口径      [100          ] A   ← text+unit  │
│  面間規格  [JIS10K       ▼]    ← select     │
│  接続      [WAFER        ▼]    ← select     │
│  ボデー材  [ADC12/EPDM    ]    ← text       │
│  シート材  [SCS13         ]    ← text       │
│  駆動      [A            ▼]    ← select     │
│  操作      [DB           ▼]    ← select     │
│                                              │
└──────────────────────────────────────────────┘
```

ラベル「弁種」は`param_def.label`から取得し、入力コントロールは`param_def.data_type`で決定する。`options`にJSON配列がある場合はドロップダウンを生成する。これにより「D01の意味を知らないと使えない」問題を根本解決する。

#### 3.4.3 バリエーションタブ

```
┌─ バリエーション ──────────────────────────────────────┐
│  軸: 塔径×高さ  刻み数: 21                            │
│                                                      │
│  ┌──────────┬────────┬────────┬────────┐            │
│  │  刻み      │ 口径    │ メーカー │ 型式    │            │
│  ├──────────┼────────┼────────┼────────┤            │
│  │ φ1200×H3500│ 65A    │ 巴      │ 700Z.. │            │
│  │ φ1400×H3500│ 80A  ▲│ 巴      │ 700Z.. │            │
│  │ φ1600×H3500│ 100A ▲│ 巴      │ 700Z.. │            │
│  │ φ1800×H3500│ 100A   │ 巴      │ 700Z.. │  ← 選択中 │
│  │ ...        │ ...    │ ...     │ ...    │            │
│  └──────────┴────────┴────────┴────────┘            │
│  ▲ = 前の刻みから変更あり                              │
└──────────────────────────────────────────────────────┘
```

## 5. データモデル

### 5.1 設計思想

**思想1: Master-Instance パターン** — 標準マスタ層と案件層を分離する。マスタは「変えない」。案件固有のカスタマイズは差分（Override）として持つ。MVPではマスタ層を実装し、案件層はテーブル定義のみ。

**思想2: Self-Describing Schema（自己記述型）** — D01〜D50の意味をリストタイプ別に定義する`param_def`マスタを設ける。UIは「L151のD02は口径なので数値入力＋A付き」とメタ情報から動的にフォームを生成できる。

**思想3: ブロック中心のバリエーション正規化** — バリエーション軸（塔径×高さ等）はブロックレベルで1回だけ定義し、各原価項目はその軸上の刻みに対して仕様値を持つ構造にする。旧Excelの「全原価項目に軸定義が重複」問題を解消する。

**思想4: D/E/F統合** — 設計条件(D列)・仕様(E列)・付帯(F列)は1つのEAVテーブルに統合する。実データ分析の結果、E列はL831で2項目、F列は0件のみであり、3テーブルに分ける必要がない。

### 5.2 ER図

```
【マスタ定義層】
  list_type_master ─────────┐
  param_def ─────────────┐  │  ← 「L151のD02=口径」を定義
                         │  │
【ブロックマスタ層】      │  │
  block_master ──────────┤  │
   ├─ variation_axis     │  │  ← ブロックのバリエーション軸
   │   └─ variation_step │  │  ← 軸上の刻み（φ1200, φ1400, ...）
   ├─ cost_item ─────────┤──┘
   │   └─ item_step_spec │      ← 原価項目×刻み＝仕様値
   │       ├─ item_param_value   ← D/E/Fパラメータ値（EAV）
   │       └─ motor_spec        ← 電動機仕様（L002等のみ）
   ├─ piping_pattern     │
   └─ block_document     │      ← PDF等の添付文書

【案件層 - Phase 2】
  project
   └─ project_block
       └─ pj_item_override      ← 案件固有の上書き
```

## 6. 機能要件

### 6.1 機器ブロックマスタ管理

#### F-001: ブロック一覧表示・検索（SCR-001）

`block_master`のブロック番号、名称、セグメント、状態で絞り込み可能な一覧画面を提供する。全文検索に加え、リストタイプやメーカー名での横断検索にも対応する。

#### F-002: ブロック新規登録

ブロック番号を自動採番（または手動入力）し、ブロック名称・セグメント等の基本情報を`block_master`に登録する。既存ブロックのコピー機能を備える。

#### F-003: ブロック編集

ブロック名称、セグメント、状態の変更を行う。

#### F-004: ブロック版管理

`block_master.revision`を管理し、draft → published → deprecated のライフサイクルを制御する。

### 6.2 原価項目管理

#### F-010: 原価項目一覧表示（左ペイン）

`cost_item`を「ツリービュー」「カテゴリービュー」「フラットリストビュー」の3モードで表示する。グルーピングは`list_type_master.category`に基づく。

#### F-011: 原価項目の追加・削除・並べ替え

`cost_item`の追加、論理削除、`sort_order`の変更を行う。原価項目コードはブロック番号＋連番で自動採番。

#### F-012: 原価項目の設計条件編集（右ペイン）

`param_def`に基づきリストタイプに応じた動的入力フォームを生成する。入力値は`item_param_value`に保存する。

#### F-013: 除外フラグ管理

`cost_item.is_excluded`を設定する。除外された項目はツリー上でグレーアウト＋取消線で表示する。

### 6.3 バリエーション管理

#### F-020: バリエーション軸定義

`variation_axis`にブロックのバリエーション軸（最大2軸）を定義する。

#### F-021: 刻み管理

`variation_step`にグループコード・刻み名・配管クラスキーを登録する。一括追加・削除・コピー機能を備える。

#### F-022: 設計条件の一括適用

主機器のバリエーション刻みに連動して、付帯機器の仕様を自動展開する。グローバルヘッダーの設計条件セレクタと連動する。

#### F-023: 予備数量の自動セット

ブロック追加時または設計条件適用時に、マスタ情報に基づいて各原価項目の`spare_qty`を自動設定する。

#### F-024: マトリクスビュー（SCR-200）

全`cost_item` × 全`variation_step`のクロスマトリクスを表示し、`item_step_spec` + `item_param_value`の仕様変更点をヒートマップで可視化する。セルのインライン編集に対応する。

### 6.4 図面・文書管理

#### F-030: ブロックフロー図表示（中央ペイン）

`block_document`（`doc_type='block_flow'`）に登録されたPDFを中央ペインのインラインビューアで表示する。拡大/縮小、パン操作、ページ切替、回転に対応する。

#### F-031: 文書の添付・削除

ドラッグ&ドロップでPDF等のファイルを`block_document`に登録する。

#### F-032: 図面-BOM双方向連動（Phase 2）

Phase 2でPDF上の機器ホットスポットとBOMツリーの双方向選択連動を実現する。

### 6.5 配管パターン管理

#### F-040: 配管パターン登録

`piping_pattern`にブロック×刻みごとの配管口径・配管クラスを登録する。

### 6.6 電動機仕様管理

#### F-050: 電動機仕様登録

回転機の`item_step_spec`に紐づく`motor_spec`の11属性を登録する。

### 6.7 BOM構造可視化

#### F-060: BOM構成サマリー

ブロック内の機器種別構成比（`list_type_master.category`別の件数）、バリエーション充足率、除外項目数をカード形式で表示する。

#### F-061: BOM整合性チェック（Phase 2）

ブロックフロー図のTAGとBOMデータの整合性を自動チェックする。

### 6.8 インポート・エクスポート（SCR-600）

#### F-070: Excelインポート

既存の機器ブロックデータ一覧表（`.xls`形式、192列×N行）をウィザード形式で取り込む。

ステップ1: ファイル選択とプレビュー。ステップ2: カラムマッピング確認（Row 0日本語ヘッダ / Row 1物理名ヘッダの二段構造に対応）。ステップ3: 変換処理。旧フラット構造を新正規化モデルに変換する際の主要ロジックは以下の通り。

`DEF_FLG=1` かつ `ITEM_GRP_CD='-'` の行 → `item_step_spec`（`step_id=NULL`、デフォルト行）。`ITEM_GRP_CD`が`!!XXXX`の行 → `variation_step`の対応する`step_id`を参照して`item_step_spec`を生成。D01〜D50 / E01〜E40 / F01〜F10の値 → `item_param_value`に展開。TANK_TYPE_CD / TYPE_NAME → `variation_axis`に1行だけ生成。

ステップ4: バリデーション結果表示＋インポート実行。

#### F-071: Excelエクスポート（Phase 2）

登録済みブロックデータをExcel形式で出力する。

#### F-072: PDF関連付け

ドラッグ&ドロップでPDFファイルを`block_document`に登録する。

## 8. 非機能要件（MVP）

### 8.1 性能要件

ブロック一覧の表示は1000件以下で1秒以内とする。1ブロック（最大900行の`item_step_spec`）の読み込み・表示は2秒以内とする。PDFビューアの初期表示は3秒以内とする。Excelインポートは1ファイル（600行程度）で10秒以内に完了する。

### 8.2 拡張性

リストタイプの追加は`list_type_master`への行追加のみで対応する。D/E/Fパラメータの追加は`param_def`への行追加のみで対応する。セグメント（業種）の追加も`block_master.segment`の値追加のみ。いずれもコード変更不要。

### 8.3 データ移行

現行の全Excelブロックデータ一覧表を初期移行対象とする。移行バッチは`F-070 Excelインポート`のロジックを再利用する。

## 9. 機器ブロックの構造仕様（参考）

### 9.1 ブロック構成の具体例

添付データ `E000000028`（重力式二層ろ過器/ポンプ移送）の分析結果に基づく。

**主機器**: 重力式二層ろ過器本体（T-GFP01 / 製缶リスト L201）

**構成**: 計39原価項目。構造物3件、回転機1件、ろ材7件、ストレーナー2件、計装品7件（うち除外1件）、自動弁7件、手動弁12件（うち除外2件）。

### 9.2 バリエーションの仕組み

**軸**: 「塔径×高さ」を第一軸として使用。φ1200〜φ7000まで21段階。高さは3500mm固定。

**連動する仕様変更の実データ**: 自動弁FV-GFP01では塔径が大きくなると口径（`D02`）が65A→80A→100A→125A→150A→200A→250A→300Aと段階的に大口径化する。メーカー（巴）と型式（700Z-7E(C)）は全刻みで共通。この「何が変わり、何が変わらないか」をマトリクスビュー（SCR-200）のヒートマップで一目で把握できるようにする。

### 9.3 D列の意味の違い（param_defの必要性の根拠）

実データ分析で確認されたリストタイプ別のD列の意味の差異:

```
L201（製缶）:   D02=塔径(φ1200)  D03=高さ(H3500)  D08=材質(SS400)
L151（自動弁）: D01=弁種(BF)     D02=口径(65A)    D03=面間規格(JIS10K)
L002（ポンプ）: D01=形式(横型渦巻) D22=ケーシング材  D23=インペラー材
L121（流量計）: D01=種類(ｵﾘﾌｨｽ)  D02=形式         D08=接続規格  D39=口径
L831（計装品）: D15=機器種別      D22=口径         E20=出力信号
```

同じ「D02」でもL201では塔径、L151では口径、L121では形式を意味する。`param_def`マスタによるメタ定義がなければ、UIはラベルを解決できず「D02: 65A」としか表示できない。

## 10. 用語定義

| 用語                 | 定義                                                             |
| -------------------- | ---------------------------------------------------------------- |
| N-BOM                | 本システムの名称。新BOMシステム                                  |
| 機器ブロック         | N-BOM第三階層。ポンプ・バルブ・計器等をまとめた標準化単位        |
| 原価項目             | 機器ブロックを構成する個々の機器・部材。第四階層に相当           |
| バリエーション軸     | ブロックが持つ設計条件の変動軸（例: 塔径×高さ）                  |
| 刻み                 | バリエーション軸上の個々のポイント（例: φ1200×H3500）            |
| リストタイプ         | 原価項目のカテゴリー（L002=ポンプ、L151=自動弁等）               |
| param_def            | D/E/F列の意味をリストタイプ別に定義するメタテーブル              |
| デフォルト行         | `item_step_spec`の`step_id=NULL`。刻み未選択時のプレースホルダー |
| 配管クラス           | 配管の材質・接合方式の分類コード（例: 1W1P）                     |
| セグメント           | 業種分類（共通/電力/土壌/医薬/海外）                             |
| 系列数               | 同一ブロックの並列設置台数。案件層で管理                         |
| 除外フラグ           | 標準構成から除外する原価項目の識別フラグ                         |
| ブロックフロー       | 機器ブロックの配管・機器配置を示す簡略P&ID                       |
| Override             | 案件固有のカスタマイズ。マスタとの差分のみ保持する方式           |
| 3ペインレイアウト    | BOMツリー＋図面ビューア＋詳細パネルの統合画面構成                |
| コンテキストスイッチ | 設計条件の選択切替による画面全体の情報更新                       |
