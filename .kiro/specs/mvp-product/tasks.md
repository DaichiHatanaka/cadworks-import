## Task 2: 自動紐付けロジックの実装

- [x] 2. 自動紐付けロジックの実装

- [x] 2.1 CAD データを起点とした自動マッチングエンジンを純粋関数として実装する
  - CAD レコードを起点に、同一 LIST_TYPE の T-BOM レコードからKID が一致するレコードを探索する
  - ID_COUNT が完全一致するペアを「保存済み（saved）」ステータスでリンク済みとして振り分ける
  - ID_COUNT の下5桁のみが一致するペアを「未保存（unsaved）」ステータスでリンク済みとして振り分ける
  - LIST_TYPE または KID が不一致のレコードは未紐付け（CAD 側・T-BOM 側それぞれ）として残す
  - 副作用なしの純粋関数として実装し、入力データの配列を受け取りマッチング結果を返す
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [x] 2.2 自動紐付けロジックのユニットテストを作成する
  - ID_COUNT 完全一致パターンのテスト
  - ID_COUNT 下5桁一致パターンのテスト
  - LIST_TYPE/KID 不一致で未紐付けとなるパターンのテスト
  - CAD に対応する T-BOM が存在しないパターンのテスト
  - T-BOM に対応する CAD が存在しないパターンのテスト
  - 複数の LIST_TYPE が混在するデータセットのテスト
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

## Task 3: 排他制御 API の実装

- [x] 3. 排他制御 API の実装

- [x] 3.1 (P) ロック取得 API を実装する
  - 工番・ユーザー情報・ロックトークンを受け取り、ロックの取得を試行するエンドポイントを作成する
  - 失効済みロック（有効期限切れ）を事前に削除してからロック取得を行う
  - PK 制約による排他保証を利用し、同一工番のロック競合時は 409 レスポンスとともにロック者情報を返す
  - ロックの有効期限（TTL）を120秒に設定する
  - _Requirements: 8.1_

- [x] 3.2 (P) ロック更新 API を実装する
  - 工番とロックトークンを受け取り、トークンが一致する場合のみ有効期限を延長するエンドポイントを作成する
  - トークン不一致時は 409 レスポンスを返す
  - _Requirements: 8.1, 8.3_

- [x] 3.3 (P) ロック解放 API を実装する
  - 工番とロックトークンを受け取り、トークンが一致する場合のみロックを削除するエンドポイントを作成する
  - _Requirements: 8.3_

- [x] 3.4 (P) ロック状態確認 API を実装する
  - 工番を指定してロック状態（ロック中かどうか、ロック者名、ロック開始日時）を返すエンドポイントを作成する
  - _Requirements: 8.2_

- [x] 3.5 失効ロックの定期クリーンアップ Cron エンドポイントを実装する
  - 有効期限切れのロックレコードを一括削除する Cron Job 用エンドポイントを作成する
  - Vercel Cron 設定を追加する
  - _Requirements: 8.3_
